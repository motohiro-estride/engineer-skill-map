# engineer-skill-map

エンジニアのスキルシートを Excel から Web に移行するための個人プロジェクト。

## プロジェクトの目的

これまで Excel で管理してきたスキルシート（職務経歴・参画プロジェクト・保有技術）を Web アプリケーションに置き換えることで、以下を実現する。

- **連携のしやすさ**: 共有・更新・参照を Web 上で完結させる（ファイル受け渡しや版管理の煩雑さを排除）
- **情報のわかりやすさ**: Excel での閲覧体験よりも整理された見せ方を実現する

## このプロジェクトを始めた動機

ユーザーは 34 歳のエンジニアで、これまでに **約 35 件** のプロジェクトを経験している。Excel に時系列で羅列されただけのスキルシートは読み手にとってつらく、Web 化して整理する必要性を感じた、というのが engineer-skill-map を立ち上げた背景。

## 配布フローと読者

スキルシートは下記フローで段階的に複数の読者に流通する:

1. **SES営業/エージェントへ提示** → 案件マッチングしてもらう
2. **客先面談担当者へ事前配布** → 面談前に把握
3. **面談本番で双方が手元に持つ** → 自分も "答えやすさ" のために参照

→ 読者は **3タイプ**: 営業/エージェント、客先面談者、自分自身。

## 配布形態

- **公開Web** (GitHub Pages) — 統合版を `/` に公開。URL 共有で閲覧可
- **Excel出力** — 同一データから生成、メール添付で渡す（営業/面談者向けの正式書類）
- **ローカルビルド** — 自分用、面談時の手元参照、メモ機能含む

## 元データ

既存の Excel スキルシートは `private/` 配下（gitignore 済）。

- `スキルシート（MM）.xlsx` — 30件のベース履歴
- `スキルシート.xlsx` — 18件の最新版（重複あり、独立件数は約32件）

## 公開プロジェクトとしての扱い

本リポジトリは **公開プロジェクト** として扱う。

- **顧客名・案件名**: 既存 Excel に **実名で書かれているもの** は契約上の承諾済（双方の会社から承諾を得ている）。マスキング不要、そのまま公開可。承諾未取得のものは Excel 上で既に "某〇〇" 表記となっている
- **本人の個人情報**: 氏名・最寄駅・勤務開始可能日・生年月日 は Web 非表示、Excel 出力時のみ含める運用
- **実データそのもの**: `private/` 配下に置き、コミットしない (gitignore 済)。公開ビルドは `private/` を読まない設計
- 環境変数・認証情報は `.env` などで管理し、コミットしない

詳細な表示レベル仕様は [.claude/design/data-model.md](design/data-model.md) 参照。

### 主要な技術選択 (詳細は tech-stack.md)

- **yarn** + Astro + Tailwind CSS + shadcn/ui
- データは YAML (1案件 = 1ファイル、`public:` / `excelOnly:` / `selfOnly:` のセクション分け)
- 公開Web は `gh-pages` ブランチへローカルから手動 push (CI 不使用)
- ビルド時に漏洩チェックスクリプトを必ず実行
